NAME

ach - manipulate ach channels  

SYNOPSIS

ach [OPTION...]  

DESCRIPTION

General tool to interact with ach channels  

OPTIONS

-C CHANNEL-NAME,
Create a new channel
-U CHANNEL-NAME,
Unlink (delete) a channel
-D CHANNEL-NAME,
Dump info about channel
-1,
With -C, accept an already created channel
-F CHANNEL-NAME,
Print filename for channel (Linux-only)
-m MSG-COUNT,
Number of messages to buffer
-n MSG-SIZE,
Nominal size of a message
-o OCTAL,
Mode for created channel
-t,
Truncate and reinit newly create channel (use only with -C). WARNING: this will clobber processes Currently using the channel.
-v,
Make output more verbose
-?,
Give this help list
-V,
Print program version
 

EXAMPLES

ach -C foo
Create channel 'foo' with default buffer sizes.
ach -C foo -m 10 -n 1024
Create channel 'foo' which can buffer up to 10 messages of nominal size 1024 bytes. Bigger/smaller messages are OK, up to 10*1024 bytes max (only one message of that size can be buffered at a time).
ach -U foo
Remove channel 'foo'
ach -1C foo
Create channel 'foo' unless it already exists, in which case leave it alone.
ach -C foo -o 600
Create channel 'foo' with octal permissions '600'. Note that r/w (6) permission necessary for channel access in order to properly synchronize.
